   - This reads from SwarmState which is populated by `merit_loader` at session start (merit_loader already reads from DB — extend it to also load `evolution_suspended` flags in a future micro-task if needed, OR have memory_writer check the DB directly)

   **Decision point:** Since memory_writer is an in-graph node and merit_loader already queries `agent_merit_scores`, the cleanest approach is to have `merit_loader` also load `evolution_suspended` flags into a new SwarmState field `evolution_suspended_agents: Dict[str, bool]`. However, to keep this plan minimal and avoid touching merit_loader, memory_writer should query the DB directly for the suspension flag:

   ```python
   async def _check_evolution_suspended(handle: str) -> bool:
       """Check if agent evolution is suspended via DB query. Non-blocking on failure."""
       try:
           pool = get_pool()
           async with pool.connection() as conn:
               result = await conn.execute(
                   "SELECT evolution_suspended FROM agent_merit_scores WHERE soul_handle = %s",
                   (handle,),
               )
               row = await result.fetchone()
               if row and row[0]:
                   return True
       except Exception as e:
           logger.debug("memory_writer: could not check evolution_suspended for %s: %s", handle, e)
       return False
   ```

2. Modify `memory_writer_node` (async function) to check suspension before calling `_process_agent`:
   ```python
   async def memory_writer_node(state: dict) -> dict:
       for handle in ALL_SOUL_HANDLES:
           try:
               if await _check_evolution_suspended(handle):
                   logger.warning("memory_writer skipped due to evolution_suspended: %s", handle)
                   continue
               _process_agent(handle, state)
           except Exception as e:
               logger.error("memory_writer: unhandled error for %s: %s", handle, e)
       return {}
   ```
