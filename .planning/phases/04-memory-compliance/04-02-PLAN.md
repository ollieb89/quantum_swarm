<objective>
Fix the ChromaDB module-level singleton in `src/tools/knowledge_base.py`.

The current file ends with `kb = KnowledgeBase()` at module scope (line 69). `KnowledgeBase.__init__`
calls `chromadb.PersistentClient()` immediately, which:
- Fails with `ModuleNotFoundError` in test environments where chromadb is not installed
- Fails with filesystem errors if `data/chroma_db` does not exist
- Cannot be tested without a real ChromaDB store

The correct pattern (established in `src/memory/service.py`) is lazy initialisation: the
ChromaDB client is created inside `__init__` on first use, not at module import time. Wrap
this with a module-level getter function `get_kb()` so callers that already import `kb`
directly can be migrated to `get_kb()` without breaking the `knowledge_base_node`.

Purpose: Eliminate the one anti-pattern explicitly flagged in the Phase 4 research doc.
Importing the module in tests must be safe without chromadb installed.

Output: `src/tools/knowledge_base.py` with lazy init and `get_kb()` getter.
Any caller currently using the `kb` singleton is updated to call `get_kb()` instead.
</objective>

```python
_svc: Optional[MemoryService] = None

def get_memory_service() -> MemoryService:
    global _svc

        _svc = MemoryService()
    return _svc
```

```python
class KnowledgeBase:
    def __init__(self):
        import chromadb                          # import inside __init__, not at top

        self.chroma_client = chromadb.PersistentClient(path=CHROMA_PATH)
```

    Make two targeted changes to `src/tools/knowledge_base.py`:

    **Change 1 — Move heavy imports inside `__init__`:**
    Remove `import chromadb`, `from chromadb.utils import embedding_functions`, and
    `import duckdb` from the top of the file. Add them as local imports at the top of
    `KnowledgeBase.__init__()`. The `logging` import stays at module level (it is stdlib).

    **Change 2 — Replace the module-level singleton with a lazy getter:**
    Remove:
    ```python
    kb = KnowledgeBase()
    ```

    Replace with:
    ```python
    _kb: "KnowledgeBase | None" = None

    def get_kb() -> "KnowledgeBase":
        """Return the module-level KnowledgeBase singleton, creating it on first call."""
        global _kb
        if _kb is None:
            _kb = KnowledgeBase()
        return _kb
    ```

    Do not change the `KnowledgeBase` class methods (`query_sentiment_context`,
    `query_historical_stats`, `__del__`). Only the import placement and singleton pattern change.

    import to `from src.tools.knowledge_base import get_kb`.
