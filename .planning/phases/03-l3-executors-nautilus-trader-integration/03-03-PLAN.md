    1. Check `risk_approved = state.get("risk_approved", False)`. If False: return `{"execution_result": {"success": False, "order_id": None, "reason": "risk_not_approved"}, "messages": [...]}`

       - `execution_mode == "live" and asset_class == "equity"`: call `await _execute_live_equity(symbol, side, quantity)`
       - `execution_mode == "live" and asset_class in ("crypto", "futures")`: call `await _execute_live_crypto(symbol, side, quantity)`
    5. Return: `{"execution_result": result_dict, "messages": [{"role": "assistant", "content": f"OrderRouter: {symbol} {side} x{quantity} via {execution_mode} — success={result_dict['success']}"}]}`

    **`async def _execute_paper(symbol, side, quantity) -> dict`:**
    Use NautilusTrader's `BacktestEngine` with a SimulatedExchange to submit and fill a market order in paper mode. Alternatively, for paper mode simplicity, simulate a fill with realistic slippage (0.01% of last price from yfinance). Return: `{"success": True, "order_id": f"PAPER-{uuid.uuid4().hex[:8].upper()}", "execution_price": price, "mode": "paper", "slippage_pct": 0.01}`. Use `asyncio.to_thread` if yfinance is called.

    Build an IB TradingNode using the pattern from RESEARCH.md Pattern 5. Read IB config from environment variables: `IB_ACCOUNT_ID`, `IB_HOST` (default "127.0.0.1"), `IB_PORT_PAPER` (default 7497).
