  <files>src/core/drift_eval.py, tests/core/test_drift_eval.py</files>
  <behavior>
    - parse_drift_guard_yaml("") returns empty tuple (no YAML block)
    - parse_drift_guard_yaml with valid YAML returns tuple of DriftRule frozen dataclasses
    - parse_drift_guard_yaml with malformed YAML raises ValueError
    - parse_drift_guard_yaml with duplicate flag_id raises ValueError
    - parse_drift_guard_yaml with unknown rule type raises ValueError
    - parse_drift_guard_yaml with keyword_ratio missing threshold raises ValueError
    - parse_drift_guard_yaml with invalid regex raises ValueError
    - DriftRule is frozen (immutable) and has fields: flag_id, type, pattern, include, threshold
    - evaluate_drift((), "any text") returns [] (no rules = no flags)
    - evaluate_drift with keyword_ratio rule: returns flag_id when ratio >= threshold
    - evaluate_drift with keyword_ratio rule: returns [] when ratio < threshold
    - evaluate_drift with keyword_any rule: returns flag_id when any term present (case-insensitive)
    - evaluate_drift with keyword_any rule: returns [] when no terms present
    - evaluate_drift with regex rule: returns flag_id when pattern matches
    - evaluate_drift with regex rule: returns [] when pattern does not match
    - evaluate_drift with multiple rules: returns list of all matching flag_ids
    - keyword_ratio and keyword_any matching is case-insensitive
    - regex runs against original text (not lowercased) unless (?i) flag used
    - "evaluation_failed" is not a valid flag_id (validation rejects it)
  </behavior>
  <action>
    Create `src/core/drift_eval.py` with:

    1. **DriftRule frozen dataclass**:
       - `flag_id: str` — slug identifier (lowercase, underscores)
       - `type: str` — one of {"keyword_ratio", "keyword_any", "regex"}
       - `pattern: str | None = None` — regex pattern string (for type=regex)
       - `include: tuple[str, ...] = ()` — keyword list (for keyword_ratio and keyword_any)
       - `threshold: float | None = None` — ratio threshold (for keyword_ratio)

    2. **parse_drift_guard_yaml(soul_text: str) -> tuple[DriftRule, ...]**:
       - Scan soul_text for a fenced code block (```yaml ... ```) within `## Drift Guard` section
       - Extract the YAML, parse with `yaml.safe_load`
       - If no YAML block found, return empty tuple
       - Validate: unique flag_ids, type in SUPPORTED_TYPES, keyword_ratio requires threshold, regex compiles, "evaluation_failed" rejected as flag_id
       - On validation failure, raise ValueError (fail-fast at load time per CONTEXT.md decision)
       - Return tuple of frozen DriftRule instances

    3. **evaluate_drift(drift_rules: tuple[DriftRule, ...], text: str) -> list[str]**:
       - If no rules, return []
       - For each rule, evaluate against text:
         - `keyword_ratio`: lowercase text, split on whitespace, count tokens matching any term in `include` (lowercased), divide by total tokens. Flag if ratio >= threshold
         - `keyword_any`: lowercase text, flag if ANY lowercased term from `include` appears as substring
         - `regex`: compile pattern, search against original text. Flag if match found
       - Return list of matched flag_id strings

    4. **SUPPORTED_TYPES frozenset** = {"keyword_ratio", "keyword_any", "regex"}

    Create `tests/core/test_drift_eval.py` with tests covering all behaviors listed above. Use pytest parametrize for rule type variants. No LLM calls, no file I/O — pure unit tests.
  </action>
