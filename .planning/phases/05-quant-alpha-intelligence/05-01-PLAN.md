<objective>
Close the two spec gaps between the CONTEXT.md decisions and the current implementation of `quant_alpha_intelligence.py`.

Gap 1: RSI output must include a `state` field (`overbought`/`oversold`/`neutral`) alongside the raw value. The current implementation returns a plain float from `handle()`.

Gap 2: The CONTEXT.md defines two formal error codes — `INSUFFICIENT_DATA` (series too short) and `INVALID_INPUT` (bad params, missing required series). The current `handle()` uses `INVALID_PARAMETER` for all ValueError cases, which conflates both error types.

Purpose: Both gaps matter for downstream consumers. Phase 6's OrderRouter calls `calculate_indicators` for ATR; QuantModeler's system prompt instructs it to use the RSI state field. Agents receiving `INVALID_PARAMETER` instead of `INSUFFICIENT_DATA` cannot distinguish a fixable data problem from a configuration error.

Output: Updated `quant_alpha_intelligence.py` and `tests/test_quant_alpha_intelligence.py` with 11 passing tests.
</objective>
