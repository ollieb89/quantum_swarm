Fix MC-01 — src/agents/rule_generator.py, persist_rules() starting at line 102:

CURRENT (lines 106-107):
```python
for rule in rules:
    self.registry.add_rule(rule)
```

TARGET — add update_status call immediately after add_rule():
```python
for rule in rules:
    self.registry.add_rule(rule)
    self.registry.update_status(rule.id, "active")
```

Rationale: The RuleValidator backtest (called at line 119) requires a live NautilusTrader environment
that is unavailable in most runs. Rules must be promoted to 'active' at persist time so they are
immediately useful; the validator can later demote them to 'rejected' if live backtests run and fail.
The update_status() method already exists (src/core/memory_registry.py line 79) and enforces the
proposed -> active transition via VALID_TRANSITIONS.

Fix MC-02 — src/graph/agents/analysts.py, MacroAnalyst() at line 133 and QuantModeler() at line 191:

CURRENT MacroAnalyst (line 133):
```python
result = _get_macro_agent().invoke({"messages": [HumanMessage(content=query)]})
```

TARGET MacroAnalyst:
```python
# Extract institutional memory message from parent state (injected by orchestrator._load_institutional_memory())
# state["messages"] is a list of dicts: [{"role": "system", "content": "INSTITUTIONAL MEMORY: ..."}]
# Convert to HumanMessage so ReAct sub-graph accepts it (sub-graphs don't process raw dicts)
_memory_msgs = state.get("messages", [])
_memory_prefix = []
if _memory_msgs:
    first = _memory_msgs[0]
    content = first.get("content", "") if isinstance(first, dict) else getattr(first, "content", "")
    if content:
        _memory_prefix = [HumanMessage(content=content)]

result = _get_macro_agent().invoke({"messages": _memory_prefix + [HumanMessage(content=query)]})
```

Apply same pattern to QuantModeler at line 191.

Important: state["messages"] uses Annotated[List[dict], operator.add] reducer — entries are dicts
({"role": str, "content": str}) not LangChain message objects. The sub-graph expects LangChain
message objects, so convert via HumanMessage(content=...). Using HumanMessage (not SystemMessage)
for compatibility with the ReAct agent's message handling.

Note on test isolation: existing test_analysts.py tests call MacroAnalyst(_make_state()) where
state["messages"] = []. The fix must handle empty messages gracefully (no memory prefix = no crash).
