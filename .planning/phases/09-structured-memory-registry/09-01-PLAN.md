    - update_status("mem_0001", "active") on a proposed rule → rule.status == "active", rule.version == 2, rule.updated_at refreshed
    - update_status("mem_0001", "deprecated") on an active rule → rule.status == "deprecated", rule.version incremented
    - update_status("nonexistent_id", "active") → raises ValueError("Rule not found: nonexistent_id")
    - update_status("mem_0001", "proposed") when already active → raises ValueError (no reverse transitions)
    - Terminal states (deprecated, rejected) cannot transition to any other status
    - save() writes to a .tmp file then os.replace() to the final path (atomic)
  </behavior>
  <action>
Add update_status() method to MemoryRegistry. Implement these rules:
1. Find rule by id; if not found, raise ValueError(f"Rule not found: {rule_id}").
2. Define VALID_TRANSITIONS dict:
   - "proposed": ["active", "rejected"]
   - "active": ["deprecated", "rejected"]
   - "deprecated": []   # terminal
   - "rejected": []     # terminal
3. If new_status not in VALID_TRANSITIONS[rule.status], raise ValueError(f"Invalid transition: {rule.status} -> {new_status}").
4. Set rule.status = new_status, increment rule.version, set rule.updated_at = datetime.now(timezone.utc).
5. Call self.save() to persist.
6. Log: logger.info("Rule %s transitioned %s -> %s (v%d)", rule_id, old_status, new_status, rule.version).

Harden save() with atomic writes:
- Write JSON to self.file_path.with_suffix('.tmp') first.
- Use os.replace(tmp_path, self.file_path) — atomic on POSIX (rename is atomic).
- Import os at top of file.

Do NOT change the existing method signatures for add_rule(), get_active_rules(), get_rule(), or _load().
