  <action>
    Create src/tools/verification_wrapper.py with:

    1. BudgetedTool class that wraps any @tool function:
       - Tracks call count per researcher invocation (passed via state context)
       - Raises ToolBudgetExceeded (custom exception) after N calls (default 5)
       - Accepts hypothesis: str as a mandatory kwarg; raises ValueError if missing
       - Logs the hypothesis to the state accumulator on each call

    2. ToolCache dict (module-level) keyed by (tool_name, frozenset(args)):
       - On cache hit: return cached result without calling the underlying tool
       - On cache miss: call tool, store result, return it
       - This reuses analyst-fetched data when researchers make identical calls

    3. budgeted(tool_fn, max_calls=5) factory function that returns a BudgetedTool wrapping tool_fn.

    Keep this module stateless at the class level — per-invocation state (call count) lives in the BudgetedTool instance passed per graph invocation.
  </action>
