CHANGE 1 — src/agents/rule_generator.py, persist_rules() (lines 102–126)

CURRENT (lines 111-113 approximately):
```python
for rule in rules:
    self.registry.add_rule(rule)
    self.registry.update_status(rule.id, "active")   # ← REMOVE THIS LINE
```

TARGET (remove only the update_status line; all other lines unchanged):
```python
for rule in rules:
    self.registry.add_rule(rule)
```

Rationale: add_rule() already saves the rule as "proposed" (MemoryRule default status).
The validator called at the end of persist_rules() reads the registry fresh (via
`self.registry.schema = self.registry._load()`) and must find proposed rules.
Removing update_status() from the loop is the only change needed in production code.

IMPORTANT — do not change any other line in persist_rules(). The validator wiring at
the end of the method (lines 123-126) is correct and must remain:
```python
validator = RuleValidator()
validator.registry = self.registry
validator.validate_proposed_rules()
```
