## Gap Root Cause and Fix

### Gap — MEM-06: asyncio.run() Nested Event Loop Failure

**Location:** `src/agents/rule_validator.py` lines 80 and 83

**Root cause:** `validate_proposed_rules()` is a synchronous method that calls `asyncio.run(asyncio.to_thread(...))` twice (lines 80, 83) to run NautilusTrader backtests. When invoked through `SelfLearningPipeline.run_review_async()` (an async coroutine), a LangGraph event loop is already running. Python 3.12 raises `RuntimeError: This event loop is already running` on the second `asyncio.run()` call. The broad `except Exception` block at lines 90-94 catches this silently, logs a warning, and leaves the rule as `proposed`. The loop continues; zero rules are promoted; no audit event is written.

**Phase 14 tests missed this** because all Phase 14 tests call `persist_rules()` directly from synchronous test functions — no running event loop exists, so `asyncio.run()` succeeds.

**Fix options (smallest diff first):**
```python
# Option A — Use ThreadPoolExecutor (no asyncio dependency)
import concurrent.futures
with concurrent.futures.ThreadPoolExecutor() as pool:
    baseline = pool.submit(_run_nautilus_backtest, base_config).result()
    treatment = pool.submit(_run_nautilus_backtest, treat_config).result()

# Option B — Make validate_proposed_rules() async (larger refactor)
async def validate_proposed_rules(self): ...
# Then update call in rule_generator.py to await it
```

**Requirements closed when fixed:** MEM-06 (fully closes). MEM-05 lifecycle transitions will complete. EXEC-04 `applied_rule_ids` will be populated. MEM-04 registry will accumulate promoted/rejected records.

## Conclusion

Milestone v1.2 has **1 remaining critical gap** after Phase 13 and Phase 14 gap closures:

- **MEM-06 async event loop conflict** — `validate_proposed_rules()` calls `asyncio.run()` inside a running event loop. All rule validation silently fails at runtime. No rules are ever promoted from `proposed`. The 2-of-3 backtest harness, audit writes, and lifecycle transitions never execute. Phase 14 tests pass because they run synchronously. Fix requires replacing `asyncio.run(asyncio.to_thread(...))` with `concurrent.futures.ThreadPoolExecutor` at `rule_validator.py:80,83`.

**MEM-04, MEM-05, RISK-07, RISK-08, and EXEC-04 are satisfied.** Two additional tech debt items (risk_approved completeness, Nyquist coverage) are non-blocking.
