```

  → _load_institutional_memory() [reads active rules from JSON registry + MEMORY.md]

  → RiskManager gate (consensus_score > 0.6)

  → trade_logger_node [writes to PostgreSQL]
```

**Flow B — Self-Improvement with Rule Validation (COMPLETE — MEM-06 gate bypass advisory)**
```
main.py --review → SelfLearningPipeline.run_review_async()
  → PerformanceReviewAgent.get_recent_trade_data() [Phase 6 column names]
  → generate_drift_report() → RuleGenerator.generate_rules()
  → persist_rules():
      → registry.add_rule(rule) [status=proposed, Phase 9]
      → update_status(rule.id, "active") [Phase 12 MC-01 — promotes BEFORE validation!]
      → RuleValidator.validate_proposed_rules() [Phase 10 — get_proposed_rules() returns empty; no-ops]
→ next main.py run → _load_institutional_memory() [now finds active rules]
```

**Advisory:** Flow B wires correctly in isolation; the Phase 12 MC-01 fix (promote-before-validate) causes the Phase 10 validation harness to no-op in the integrated pipeline. Fix: reorder persist_rules() to validate before promoting.
